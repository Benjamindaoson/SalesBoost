# SalesBoost 项目优化路线图

本路线图基于项目现状审计与长远发展规划制定，旨在将 SalesBoost 从 MVP 原型打磨为成熟的生产级产品。

## 1. 项目现状评估 (Status Assessment)

*   **当前状态**: 后端核心（FSM/RAG）成熟，前端处于 Mock 演示阶段，文档齐全。
*   **关键差距**: 前后端数据未打通，缺乏生产级监控，用户权限体系缺失。

## 2. 技术优化方向 (Technical Optimization)

### 短期目标 (v1.1) - **进行中**
- [ ] **前端去 Mock 化**: 封装 HttpClient，全面对接后端真实 API。
- [ ] **测试覆盖率提升**: 引入 Jest/Vitest，重点补充前端业务逻辑测试。
- [ ] **CI/CD 增强**: 完善 GitHub Actions，增加前端构建检查。

### 中期目标 (v1.2)
- [ ] **架构重构**: 将单体应用向模块化演进，分离 `Core Engine` 与 `API Service`。
- [ ] **性能优化**: 引入 Redis 缓存层，优化 RAG 检索响应速度 (<1s)。

## 3. 功能完善计划 (Feature Enhancement)

### 核心功能补全
- [ ] **用户权限体系 (RBAC)**: 实现管理员/教练/学员三级权限控制。
- [ ] **数据大屏 (Dashboard)**: 从静态展示升级为实时数据流，支持多维度下钻分析。
- [ ] **语音交互**: 集成 STT/TTS，支持纯语音对练模式。

### 体验优化
- [ ] **引导式教学**: 新用户首次进入时的 Step-by-Step 引导。
- [ ] **移动端适配**: 优化 H5 体验，支持随时随地练习。

## 4. 文档体系建设 (Documentation)

- [x] **架构设计文档**: 已完成 (`ARCHITECTURE.md`).
- [ ] **API 接口文档**: 需自动生成并托管 Swagger UI。
- [ ] **部署运维手册**: 完善容器化部署与故障排查指南。
- [ ] **用户操作手册**: 编写面向最终用户的图文教程。

## 5. 性能优化措施 (Performance)

- [ ] **基准测试**: 建立 JMeter/Locust 压测脚本，确定并发瓶颈。
- [ ] **数据库调优**: 对核心查询字段添加索引，优化 PostgreSQL 配置。
- [ ] **监控告警**: 接入 Prometheus + Grafana，监控 QPS、延迟及错误率。

## 6. 团队协作优化 (Team Collaboration)

- [ ] **规范化流程**: 确立 Git Flow 分支管理策略。
- [ ] **代码审查**: 强制 Pull Request 机制，引入自动化 Code Review 工具。
- [ ] **知识库**: 建立内部 Wiki，沉淀技术难题解决方案。

## 7. 长期维护规划 (Long-term Maintenance)

- [ ] **技术债管理**: 每月预留 20% 时间用于重构和依赖升级。
- [ ] **定期巡检**: 每季度进行一次全面的安全与性能审计。
- [ ] **技术升级**: 持续关注 LLM 模型发展，适时升级基座模型 (如 GPT-5, Claude 4 等)。
