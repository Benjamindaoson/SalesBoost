# MVP 验收口径 - 8条E2E用例

## 用例1：权益问答 - 返回带来源的短回复

### 前置条件
- 会话已创建并连接 WebSocket
- 知识库中已有权益/活动信息

### 测试步骤
1. 用户发送："这张卡有什么权益？"
2. 系统通过 WebSocket 返回 `turn_result` 消息
3. 检查 `quick_suggest` 字段

### 预期结果
```json
{
  "quick_suggest": {
    "intent_label": "权益问答",
    "suggested_reply": "这张卡主要权益包括机场贵宾厅、酒店升级、积分兑换等。",
    "alt_replies": [
      "主要权益有：机场贵宾厅无限次使用、全球酒店升级、积分永久有效。",
      "权益包括机场贵宾厅、酒店升级、积分兑换，特别适合经常出差的您。"
    ],
    "confidence": 0.85,
    "evidence": {
      "source_titles": ["信用卡权益说明", "官方活动规则"],
      "source_snippets": ["机场贵宾厅...", "酒店升级..."]
    }
  }
}
```

### 验证点
- ✅ `suggested_reply` <= 220字符
- ✅ `alt_replies` >= 2条
- ✅ `evidence.source_titles` 不为空
- ✅ 前端显示意图标签"权益问答"
- ✅ 点击"复制发送"按钮，文本填入输入框

## 用例2：异议处理 - 返回可复制话术 + 换一句可切换

### 前置条件
- 会话进行中，客户提出异议

### 测试步骤
1. 用户发送："年费太贵了"
2. 系统返回快速建议
3. 前端显示建议面板
4. 点击"换一句"按钮

### 预期结果
- ✅ `intent_label` = "异议处理"
- ✅ `suggested_reply` 包含异议处理话术（如"我理解您对价格的考虑..."）
- ✅ `alt_replies` 包含至少2条不同表述
- ✅ 点击"换一句"后，显示下一条备用话术
- ✅ 循环切换，最后回到第一条

### 验证点
- ✅ 话术可直接复制发送（不包含"你可以..."等教学性表述）
- ✅ 切换流畅，无闪烁

## 用例3：推进成交 - 识别推进时机并给推进句

### 前置条件
- 会话进行中，客户表达积极信号

### 测试步骤
1. NPC 回复："听起来不错，我再考虑一下"
2. 用户发送："好的，没问题"
3. 系统识别推进时机

### 预期结果

```json
      "没问题，我马上为您处理。您还有其他需要了解的吗？"

  }
}
```

### 验证点
- ✅ 识别推进时机准确
- ✅ 推进话术明确、可执行

## 用例4：输入违规承诺 - BLOCK + 给安全改写 + 发送被禁用

### 前置条件
- 用户在输入框输入内容

### 测试步骤
1. 用户在输入框输入："绝对能保证您稳赚不赔"
2. 等待 500ms debounce
3. 系统调用合规检测 API
4. 检查前端显示

### 预期结果
```json
{
  "risk_level": "BLOCK",
  "risk_tags": ["exaggeration", "misleading"],
  "safe_rewrite": "根据过往经验，大多数情况下可以满足您的需求，但任何投资都有风险。",
  "reason": "这句话有合规风险，建议这样说："
}
```

### 验证点
- ✅ 输入框下方显示红色警告框
- ✅ 显示"发送被禁用"
- ✅ 显示安全改写版本
- ✅ 发送按钮被禁用（disabled）
- ✅ 点击"一键替换"后，输入框内容替换为 `safe_rewrite`
- ✅ 合规事件写入 `compliance_logs` 表

## 用例6：会话结束 - 弹出<=3条复盘反馈

### 前置条件
- 会话进行多轮对话
- 触发会话完成（阶段=COMPLETED 或用户主动结束）

### 测试步骤
1. 会话完成
2. 系统生成轻量复盘
3. 前端弹出反馈卡片

### 预期结果
```json
{
  "type": "session_complete",
  "micro_feedback": {
    "feedback_items": [
      {
        "title": "异议处理后可推进",
        "what_happened": "您识别了客户的异议，但没有及时推进成交",
        "better_move": "在化解异议后，可以尝试推进：'既然您已经了解了权益，不如现在就办理，还能享受当前优惠'",
        "copyable_phrase": "既然您已经了解了权益，不如现在就办理，还能享受当前优惠"
      },
      {
        "title": "抓住推进时机",
        "what_happened": "客户表达了积极信号，但您没有及时给出明确的下一步",
        "better_move": "当客户表示同意时，立即给出具体行动：'太好了！我现在就帮您提交申请，请确认一下您的信息'",
        "copyable_phrase": "太好了！我现在就帮您提交申请，请确认一下您的信息"
      }
    ],
    "total_turns": 8
  }
}
```

### 验证点
- ✅ 反馈项 <= 3条
- ✅ 每条包含 `copyable_phrase`（可直接复用）
- ✅ 点击"复制句子"按钮，文本复制到剪贴板
- ✅ 不打分、不排名、不画雷达图
- ✅ 反馈保存到 `session_feedbacks` 表

## 用例7：日志落库 - 能查到合规事件与反馈

```sql
SELECT * FROM session_feedbacks WHERE session_id = 'xxx';
```

- ✅ `total_turns` > 0

## 用例8：降级策略 - RAG无来源时不胡编，输出安全兜底文案

### 前置条件
- 知识库中无相关权益信息

### 测试步骤
1. 用户发送："这张卡有什么特殊权益？"（知识库中不存在）
2. 系统 RAG 检索无结果
3. 检查快速建议

### 预期结果
```json
{
  "quick_suggest": {
    "intent_label": "权益问答",
    "suggested_reply": "关于权益详情，建议您查看官方规则或联系客服确认，以确保信息准确。",
    "alt_replies": [
      "具体的权益信息，我建议您查看我们的官方说明，这样更准确。",
      "为了确保信息的准确性，建议您通过官方渠道查询最新的权益详情。"
    ],
    "confidence": 0.5,
    "evidence": null
  }
}
```

### 验证点
- ✅ `evidence` 为 `null`（无来源）
- ✅ `suggested_reply` 不包含具体权益描述（避免编造）
- ✅ 引导用户查看官方渠道
- ✅ `confidence` 较低（0.5）
